Training via Python API

If you prefer to train your model programmatically, use this standalone script to reproduce the KalEdge workflow. This script loads the scientific dataset, builds the quantized model, and prepares the HLS configuration.

import pandas as pd
import tensorflow as tf
import qkeras
import hls4ml
import matplotlib.pyplot as plt
from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay

# 1. Load Data
df = pd.read_csv('/app/assets/gamma_neutron.csv')
X = df.drop('class', axis=1).values
y = tf.keras.utils.to_categorical(df['class'])

# 2. Build QKeras Model
model = tf.keras.Sequential([
    tf.keras.layers.Flatten(input_shape=(X.shape[1],)),
    qkeras.QDense(16, kernel_quantizer="quantized_bits(8,2)"),
    qkeras.QActivation("quantized_relu(8)"),
    qkeras.QDense(8, kernel_quantizer="quantized_bits(8,2)"),
    qkeras.QActivation("quantized_relu(8)"),
    qkeras.QDense(y.shape[1], activation="softmax")
])

# 3. Train
model.compile(optimizer='adam', loss='categorical_crossentropy', metrics=['accuracy'])
history = model.fit(X, y, epochs=32, batch_size=32, validation_split=0.2)

# 4. Visualize Results (Exercise)
# TODO: Implement plotting for training curves (accuracy/loss history)
# and the Confusion Matrix to evaluate model performance.

# 5. Convert to hls4ml
config = hls4ml.utils.config_from_keras_model(model, granularity='name')
config['Model']['Precision'] = 'ap_fixed<16,6>'
config['Model']['ReuseFactor'] = 8
config['Model']['Strategy'] = 'Latency'

hls_model = hls4ml.converters.convert_from_keras_model(
    model,
    hls_config=config,
    output_dir='hls4ml_prj',
    part='xczu3eg-ffvb1156-1-e' # Ultra96-V2
)
hls_model.build()

A Deeper Look: The DMA Wrapper

To connect your neural network to the Processing System (PS) via DMA, you need a top-level accelerator wrapper. These source files (.cpp and .h) are provided in the day-3/dma_wrapper directory of the repository. This wrapper handles the AXI Stream protocol, reading input data and writing predictions back to memory.

#include "myproject_auto_accel.h"
#include "myproject.h"

void myproject_auto_accel(
    hls::stream<axis_t> &s_axis_in,
    hls::stream<axis_t> &s_axis_out
) {
#pragma HLS INTERFACE axis      port=s_axis_in
#pragma HLS INTERFACE axis      port=s_axis_out
#pragma HLS INTERFACE s_axilite port=return bundle=CTRL

    input_t in_arr[161];
#pragma HLS ARRAY_PARTITION variable=in_arr complete dim=1

READ_IN:
    for (int i = 0; i < 161; i++) {
    #pragma HLS PIPELINE II=1
        axis_t din = s_axis_in.read();
        in_arr[i] = axis_to_fixed<16,6>(din);
    }

    result_t out_arr[2];
#pragma HLS ARRAY_PARTITION variable=out_arr complete dim=1

    myproject(in_arr, out_arr);

WRITE_OUT:
    for (int i = 0; i < 2; i++) {
    #pragma HLS PIPELINE II=1
        axis_t dout;
        fixed_to_axis<16,6>(dout, out_arr[i]);
        dout.keep = -1; dout.strb = -1;
        dout.user = 0; dout.id = 0; dout.dest = 0;
        dout.last = (i == 2 - 1);
        s_axis_out.write(dout);
    }
}

This wrapper ensures the model is hardware-ready, automating the data transfer between the FPGA logic and the ARM processor.

15

Block Design Construction

Create a new Block Design and add the Zynq UltraScale+ MPSoC. Run Block Automation with "Apply Board Presets" enabled. Then, follow these critical configurations:

  • PS-PL Configuration: Enable AXI HPM0 LPD and AXI HP0 FPD.

  • Clock Configuration: Ensure PL Fabric Clock (FCLK_CLK0) is set to 100 MHz.
  • AXI DMA: Add the DMA IP and disable Scatter-Gather. Set the Burst Size to 16.

  • MLP IP: Add your generated myproject_auto_accel IP from the repository.

Use Connection Automation to let Vivado route the AXI interconnects and reset logic automatically. IMPORTANT: The AXI-Stream data connections between the DMA and the Accelerator (Input/Output) must be performed manually. Ensure the final connections look exactly as shown in the figure below:

Bitstream and Hardware Export
